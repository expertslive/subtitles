# EventSubtitles v3.1.0

## Download

The GitHub release asset is:

```text
EventSubtitles-v3.1.0-macos-arm64.zip
```

It contains `EventSubtitles.app` for Apple Silicon Macs. The app is displayed as `Subtitles`.

SHA-256:

```text
fdfbe4bc2a4c25baaa3f8584d81a33bbab1f5d93729f22eb486eec8c3a1f9e79
```

## Highlights

- Added an app-level audio input selector in the Audio workspace.
- Wired the selected input into the audio meter, CAF recording, and WhisperKit capture path.
- Kept `System default` as a live Core Audio default instead of pinning it to a stale device ID.
- Added fallback handling when a previously selected input interface is unavailable.
- Added unit coverage for audio input selection resolution.

## Validation

Release validation commands:

```bash
swift build --product EventSubtitles
swift run EventSubtitlesCoreUnitTests
swift run EventSubtitlesSmokeTests
./scripts/build_app_bundle.sh
codesign --verify --deep --strict build/EventSubtitles.app
```

## First Launch

The app bundle is ad-hoc signed for local testing. macOS may require opening it from Finder with Control-click > Open the first time.
