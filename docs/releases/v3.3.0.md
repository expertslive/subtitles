# EventSubtitles v3.3.0

## Download

The GitHub release asset is:

```text
EventSubtitles-v3.3.0-macos-arm64.zip
```

It contains `EventSubtitles.app` for Apple Silicon Macs. The app is displayed as `Subtitles`.

SHA-256:

```text
06e6b334497423923a988de9f0edd4c9c138c82b698bfe8339abfd5f744657f7
```

## Highlights

- Migrated the operator state model to Swift Observation for narrower SwiftUI updates.
- Replaced the fixed caption polling loop with demand-driven caption ticks.
- Added a macOS Settings window for Style, Audio, Models, and Translation setup.
- Added safer app lifecycle handling, including live-session quit confirmation and awaited shutdown.
- Preserved and restored output-window placement across launches.
- Improved the audio meter, caption animations, and reduce-motion behavior.
- Fixed review issues around version/documentation carry-forward, panic blank behavior, Stop-time rolling-caption flush, CAF recording continuity after audio restart, and Swift concurrency warnings in the capture converter.

## Validation

Release validation commands:

```bash
swift build --product EventSubtitles
swift run EventSubtitlesCoreUnitTests
swift run EventSubtitlesSmokeTests
./scripts/build_app_bundle.sh
codesign --verify --deep --strict build/EventSubtitles.app
```

## First Launch

The app bundle is ad-hoc signed for local testing. macOS may require opening it from Finder with Control-click > Open the first time.
